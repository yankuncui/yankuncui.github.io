计算机操作系统

  • 操作系统最基本的特征是并发和共享，两者互为存在条件。

  • 操作系统主要向用户提供命令接口和程序接口(系统调用)，图形接口其实是调用了系统调用而实现的功能。

  • 广义指令就是系统调用指令，而命令解析器属于命令接口，shell是命令解析器。系统中的缓存全部由操作系统管理，对用户是透明的操作系统不提供管理系统缓存的系统调用。

  • 分时系统追求的目标是比较快速的响应用户。

  • 允许多个用户以交互的方式使用计算机的操作系统，称为分时操作系统；允许多个用户将若干作业提交给计算机系统中处理的操作系统称为批处理系统，在实时操作系统的控制下，计算机操作系统能及时处理由过程控制反馈的数据，并及时做出响应。

  • 实时性和可靠性是实时操作系统最重要的两个目标。

  • 中断技术使得多道批处理系统的I/O设备可与CPU并行工作。

  • 批处理操作系统的用户脱机使用计算机，作业是成批处理的，系统内多道程序并发执行，交互能力差。

  • 分时操作系统可让多个用户同时使用计算机，人机交互性较强，具有每个用户独立使用计算机的独立性，系统响应及时。

  • 实时操作系统能对控制对象做出及时反应，可靠性高，响应及时，但资源利用率低。

  • 当中断发生后，CPU立即进入核心态。

  • 当中断发生后，当前运行的进程暂停运行，并由操作系统内核对中断进行处理。

  • 用户态—>核心态是通过中断实现的，并且中断是唯一途径。

  • 核心态—>用户态的切换是通过执行一个特权指令将程序状态字(psw)的标志位设为用户态。

  • 陷阱陷入：有意为之的异常，如系统调用。

  • 故障：由错误条件引起的，可能被故障处理程序修复，如缺页。

  • 终止：不可恢复的致命错误造成的结果，终止处理程序不在将控制返回给引发终止的应用程序，如整数除0.

  • 中断机制的诞生：为了实现多道程序并发执行而引入的一项技术。

  • 外中断的处理过程：

    1. 每条指令执行技术后，CPU检查是否有外部中断信号。
    2. 如有外部中断信号，则需要保护被中断的进程、CPU环境。
    3. 根据中断信号类型转入相应的中断处理程序。
    4. 恢复原进程的CPU环境并退出中断，返回原进程继续往下执行。

  • 操作系统作为用户和计算机硬件之间的接口，需要向上提供一些简单易用的服务，主要包括命令接口和程序接口其中程序接口由一组系统调用组成。

  • 命令接口(允许用户直接使用)

    联机命令接口：用户说一句系统做一句

    脱机命令接口：用户说一堆系统做一堆

    程序接口(允许用户通过程序间接使用)：由一组系统调用组成

  • 系统调用是操作系统提供给应用程序 (程序员)使用的接口，可以理解为一种可供应用程序调用的特殊函数，应用程序可以发出系统调用请求来获得操作系统的服务。

  • 系统调用相关处理涉及到对系统资源的管理、对进程的控制，这些功能需执行一些特权指令才能完成，因此系统调用的相关处理需在核心态下执行

  • 系统调用

    1. 设备管理：完成设备的请求/释放/启动等功能
    2. 文件管理：完成文件的读/写/创建/删除等功能
    3. 进场控制：完成进程的创建/撤销/阻塞/唤醒等功能
    4. 进程通信：完成进程之间的消息传递/信号传递等功能
    5. 内存管理：完成内存的分配/回收等功能

  • 陷入指令是在用户态执行的，执行陷入指令后立即引发一个内中断，从而CPU进入核心态。

  • 发出系统调用请求是在用户态，而对系统调用的相应处理在核心态下进行。

  • 陷入指令是唯一一个只能在用户态下执行，而不可在核心态执行的指令。

  • 凡是与资源有关的操作，会直接影响到其他进程的操作，一定需要操作系统的介入，既需要通过系统调用来实现。

  • 系统调用发生在用户态，对系统调用的处理发生在核心态，执行陷入指令会产生内中断，使处理器从用户态进入核心态。

  • 执行系统调用的过程包括如下操作：传递系统调用参数—>执行陷入(trap)指令—>执行相应的服务程序—>返回用户态。

  • 外部中断处理过程中，PC值由中断隐指令自动保存，而通用寄存器内容由操作系统保存。

  • 时间中断的主要工作是处理和时间有关的信息及决定是否执行调度程序，和时间有关的所有信息包括系统时间、进程的时间片延时。使用CPU的时间，各种定时器。

  • 用户可以在用户态调用操作系统的服务，但执行具体的系统的调用服务程序是处于内核态的。

  • 系统调用是操作系统内核为应用程序提供服务的接口。

  • 当CPU检测到中断信号后，由硬件自动保存被中断程序的断点(即程序计数器PC)，之后硬件找到该中断信号对应的中断向量，中断向量指各种中断服务程序入口地址(各中断向量统一存放在中断向量表中，该表由操作系统初始化)，接下来开始执行各种中断服务程序，保存PSW、保存中断屏蔽字，保存各通用寄存器的值，并提供与中断信号对应的中断服务，中断服务程序属于操作系统内核。

  • 操作系统的引导程序位于磁盘活动分区的引导扇区中，引导程序分为两种；一种是位于ROM中的自举程序(BIOS的组成部分)用于启动具体的设备，另一种是位于装有操作系统硬盘的活动分区的引导扇区中的引导程序(称为启动管理器)用于引导操作系统。

  • BIOS控制权给排在首位的启动设备后，CPU将该设备主引导扇区的内容(主引导记录MBR)加载到内存中，然后由MBR检查分区表，查找活动分区，并将该分区的引导扇区的内容(分区引导记录的PBR)加载到内存加以执行。

  • 进程是暂时的，程序永久的，进程是动态的，程序是静态的，进程至少由代码、数据、PCB组成，程序仅需代码和数据即可；程序代码经过多次创建可对应不同进程，而同一个系统的进程(或线程)可以由系统调用的方法被不同的进程(或线程)多次使用。

  • 只有从运行态到阻塞态才由进程自身决定。

  • 在同一进程中，线程得切换，不会引起进程的切换。

  • 线程的切换，可能引起进程的切换。

  • 同一进程或不同进程内的线程都可以并发执行。

  • 只能由阻塞态到就绪态，不能由阻塞态到运行态。

  • 当进程的时间片到时，进程由运行态转变为就绪态，等待下一个时间片的到来。

  • 进程切换指调度不同的进程执行，当一个进程由运行态到就绪态，CPU调度引起进程的切换。

  • 若系统只有用户级进程，则处理机调度对象是进程。

  • 同一进程内的线程切换，需要用户态到核心态进行，系统开销较大。

  • 内核级线程的程序实体可以在内核态运行。

  • 进程中的某个用户线程被阻塞，则整个进程也被阻塞。

  • 用户线程的切换可以在用户空间完成，内核级线程的切换需要的操作系统帮助进行调度，因此用户级的线程切换效率更高。

  • 临界资源只能为一个进程所使用。

  • 多线程是指在一个程序中可以定义多个线程并同时运行他们，每个进程执行不同的任务。多线程与多任务的区别：多任务是针对操作系统而言的，代表操作系统可以同时执行的程序个数；多线程是针对一个程序而言的，代表一个程序可以同时执行的线程个数，而每个线程可以完成不同的任务。

  • 先来先服务算法有利于CPU繁忙型的作业，而不利于I/O繁忙型的作业。

  • 系统进程优先级高于用户进程，前台进程高与后台进程，操作系统更倾向于I/O进程。

  • 进程调度的时机包括：运行的进程运行完毕、运行的进程自我阻塞、运行的进程的时间片用完、运行的进程所需要的资源没有准备好、运行的进程出现错误。以上时间都在CPU方式为不可抢占方式时引起调度，在CPU是可抢占方式时，就绪队列中的某个进程的优先级高于当前运行进程的优先级时也会发生进程调度。

  • 静态优先级是在创建进程时确定的，确定之后在整个运行期间不再改变。

  • 在动态优先权中，随着进程执行时间的增加其优先权随之降低，随着作业等待时间的增加其优先权相应上升。

  • 时间片轮转调度算法是绝对可抢占的，因为它是按照时间配额来运行的，时间一到，不管是否完成，当前进程必须撤下换新的。

  • 分时系统的时间片固定，因此用户数越多，响应时间越长。

  • 能够满足短作业优先且不会发生饥饿现象的是高响应比优先算法。因为高响应比优先算法在等待时间相同的情况下，作业执行的时间越短，响应比越高，满足短任务优先，随着长作业等待时间的增加，响应比会变大，执行机会也会增大，因此不会发生饥饿现象。

  • 时间片轮转不会导致饥饿。

  • 短任务优先调度不管是抢占式还是非抢占式的，当系统总是出现新来的短任务时，长任务总得不到处理机，所以饥饿。

  • 影响时间片大小的主要因素包括响应时间，系统开销和进程数量等。

  • 阻塞队列中的进程只有被唤醒并进入就绪队列后，才能参与调度，所以该调度进程不会使用阻塞序列。

  • 进程同步是进程之间的一种协同工作关系，这里进程的并发是异步的，它们相互合作，共同完成一项任务。

  • 互斥信号量mutex的初值为1，表示允许一个进程临界区，当有一个进程进入临界区(没有进程等待进入时，mutex减1变为0，‘等不等待都要-1’),ImutexI的值为等待进入进程数。

  • 引入的多道程序技术的前提条件之一是系统具有中断功能。

  • 死锁状态一定是不安全状态。

  • 产生死锁的根本原因是系统资源分配不足和进程推进顺序非法。
